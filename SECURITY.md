# Security Policy

## Scope

This project processes source indexes and local text. It must not collect credentials, bypass access controls, or transmit a user's corpus.

## Before reporting

Do not include real tokens, private source text, personal data, or paid material in an issue. Use a redacted reproduction or synthetic fixture.

## Report privately

Report suspected secret exposure, unexpected network access, path traversal, unsafe archive handling, or destructive file behavior through [GitHub Private Vulnerability Reporting](https://github.com/zethfion/rw-master-system/security/advisories/new). Do not open a public issue containing sensitive evidence.

## Supported releases

Security fixes apply to the latest `0.2.x` release. Older development snapshots are unsupported.
