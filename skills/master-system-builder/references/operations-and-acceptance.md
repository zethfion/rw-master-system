# Operations and acceptance

## MVP acceptance

- The package has a valid manifest, README, soul, index, evidence registers, and expected directories.
- At least one source pipeline works end to end.
- Source candidates have explicit dispositions.
- Important claims link to local source anchors.
- At least three useful framework cards exist.
- Speaker identity and evidence level are recorded.
- Unknowns and access boundaries are visible.
- No credentials, temporary signed URLs, or unnecessary personal data are present.
- One realistic consultation answers with evidence and uncertainty.
- One framework-application task uses a method without replacing the host agent's identity.
- The human receives a usage and maintenance lesson.

## Coverage audit

Do not use file counts as proof of completeness. Compare exact sets:

- discovered candidates versus source-register dispositions;
- accepted sources versus corpus files;
- corpus files versus distillates or documented exclusions;
- material claims versus evidence anchors;
- named methods versus framework cards or open questions.

Record a discovery cutoff date and the legal source boundary. “Complete” must mean complete within that defined boundary.

## Incremental maintenance

1. Place new sources in a candidate queue.
2. Verify identity, rights, and duplication.
3. Normalize and register them.
4. Create distillates for the new material.
5. Update indexes.
6. Change soul or frameworks only when evidence warrants it.
7. Preserve conflict history.
8. Re-run validation and record the package version.

## Reliability

- Checkpoint large media acquisition before optional caption or enrichment work.
- Use stable provider IDs as durable keys.
- Write state atomically when scripts are introduced.
- Keep raw inputs immutable and regenerated derivatives replaceable.
- Keep runtime consultation logs outside content manifests and release comparisons.

## Failure behavior

Fail closed when:

- authorization is unclear;
- the speaker cannot be identified;
- a rule lacks sufficient evidence;
- a platform requires bypassing controls;
- a destructive overwrite or deletion is proposed;
- a secret or private corpus may enter a public repository.
